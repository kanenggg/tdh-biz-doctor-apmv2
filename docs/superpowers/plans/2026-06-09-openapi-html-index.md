# OpenAPI HTML Index Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the local OpenAPI CLI generate `openapi/index.html` beside the generated JSON/YAML specs.

**Architecture:** Keep the existing `openapi` binary as the only generator. Refactor the generator just enough to return generated artifact metadata, then write a static Swagger UI HTML index that references the active generated spec file.

**Tech Stack:** Rust 2024, clap, anyhow, serde_json, serde_yaml, utoipa, existing `just` recipes.

---

## File Structure

- Modify `cli/src/openapi.rs`: generate JSON/YAML, track generated modules, write `index.html`, and make `generate-all` fail on generation errors.
- Generated output `openapi/index.html`: created by running the CLI, not hand-maintained.
- Existing output `openapi/consultation-rs.json` and `openapi/consultation-rs.yaml`: regenerated by the CLI.
- No changes to `consultation-rs/openapi/swagger.html` or `consultation-rs/openapi/redoc.html`.

### Task 2: Generate The Static HTML Index

**Files:**
- Modify: `cli/src/openapi.rs`

- [ ] **Step 1: Add `write_openapi_index`**

Add this function below `generate_openapi`:

```rust
fn write_openapi_index(output_dir: &PathBuf, generated_specs: &[GeneratedOpenApi]) -> Result<()> {
    let primary_spec = generated_specs
        .first()
        .ok_or_else(|| anyhow::anyhow!("No OpenAPI specs were generated"))?;

    let title = format!("{} OpenAPI", primary_spec.module);
    let html = render_openapi_index_html(&title, &primary_spec.json_file_name);
    let index_file = output_dir.join("index.html");

    fs::write(&index_file, html)
        .map_err(|e| anyhow::anyhow!("Failed to write OpenAPI HTML index: {}", e))?;
    println!("  HTML: {}", index_file.display());

    Ok(())
}
```

- [ ] **Step 2: Add the HTML renderer helper**

Add this function below `write_openapi_index`:

```rust
fn render_openapi_index_html(title: &str, spec_url: &str) -> String {
    format!(
        r#"<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>{title}</title>
    <link rel="stylesheet" href="https://unpkg.com/swagger-ui-dist@5/swagger-ui.css">
    <style>
        body {{
            margin: 0;
            background: #ffffff;
        }}
        .topbar {{
            display: none;
        }}
    </style>
</head>
<body>
    <div id="swagger-ui"></div>
    <script src="https://unpkg.com/swagger-ui-dist@5/swagger-ui-bundle.js"></script>
    <script>
        window.onload = function() {{
            window.ui = SwaggerUIBundle({{
                url: "{spec_url}",
                dom_id: "#swagger-ui",
                deepLinking: true,
                layout: "BaseLayout"
            }});
        }};
    </script>
</body>
</html>
"#
    )
}
```

- [ ] **Step 3: Add a focused renderer test**

Add this test module at the bottom of `cli/src/openapi.rs`:
