# Booking conflicts use doctor occupancy windows

Scheduled and instant bookings share the same doctor occupancy truth. We decided conflict checks must use doctor time-window overlap, not slot-id equality, and active reservations/bookings for one doctor must not overlap. All consultations include post-consultation buffer in the consultation slot by default. For scheduled booking, the occupied window equals the booked/prepaid slot window; no second hidden buffer is added after the scheduled slot because the schedule cadence already accounts for that buffer. For instant booking, the doctor's occupied window also includes a pre-payment buffer before consultation start so doctor capacity is held while the patient completes payment. If a consultation session ends before the planned occupied-until time, occupancy may be shortened to the actual session end; a subsequent instant booking may start inside the remainder of a previous scheduled slot and may overlap multiple scheduled slot labels, because conflicts are based on actual time windows.
